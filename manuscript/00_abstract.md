# Abstract

**Background.** Whether hospitals are located where people need them is a
long-standing question in health geography, usually approached with
catchment-based accessibility scores (e.g., two-step floating catchment area)
and residential census population. We instead pose the question as a formal
spatial point-process hypothesis test and ask it against *ambient* population —
where people actually are over a 24-hour day.

**Methods.** We test whether U.S. hospital locations follow population using an
**inhomogeneous Ripley's K-function** in which the null intensity is set
proportional to population, evaluated against a Monte-Carlo envelope of
inhomogeneous-Poisson simulations. Hospital locations come from HIFLD
(*n* = 7,966 open facilities); population is ORNL LandScan Global **ambient**
(24-hour) population at ~1 km, contrasted against NASA SEDAC GPWv4.11
**residential** population at the same ~1 km grid. The analysis is run
independently within each U.S. **commuting zone** (county clusters that tile
the entire country, rural included), with a Myllymäki global envelope test for
family-wise error control across zones. We additionally weight by hospital
capacity (staffed beds) and restrict to trauma centers.

**Results.** *(Pending the national run — to be completed.)* We report the
fraction of CBSAs in which hospital clustering significantly exceeds what
population predicts, the geography of over-concentration versus under-served
deviation, and the extent to which conclusions change when the ambient
population null is replaced by the residential null.
